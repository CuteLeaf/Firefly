# 内容发布模板与实施清单

## 一、内容发布模板文档

这份文档的目标是统一后续发内容的方式，避免栏目越来越乱。

### 1. 通用 Frontmatter 模板

所有内容建议先使用这套基础模板：

```yaml
---
title: 标题
published: 2026-06-10
description: 一句话简介
category: 日常
tags: [生活, 记录]
author: 你的名字
draft: false
comment: true
image: ""
---
```

字段说明：

- `title`：文章标题
- `published`：发布日期
- `description`：摘要
- `category`：栏目名称
- `tags`：细分类标签
- `author`：作者
- `draft`：是否草稿
- `comment`：是否允许评论
- `image`：封面图，可留空

如果是私密内容，再增加：

```yaml
password: 你们约定的密码
passwordHint: 只有对方能看懂的提示
```

### 2. 日常内容模板

适合记录当天的小事、情绪、见面、纪念日。

```md
---
title: 今天想记一下
published: 2026-06-10
description: 关于今天的一点小记录
category: 日常
tags: [生活, 心情]
author: A
draft: false
comment: true
---

今天发生了什么。

我最想记住的一个细节是什么。

我此刻的心情是什么。

如果这是写给对方看的，我最想让对方知道的一句话是什么。
```

建议标题：

- `今天想记一下`
- `这个周末过得很慢`
- `和你见面的那一天`
- `最近的一点生活更新`

### 3. 推荐内容模板

适合书、小说、电视剧、电影、音乐等。

```md
---
title: 想推荐给你的一本书
published: 2026-06-10
description: 这本书让我第一时间想到你
category: 推荐
tags: [书, 推荐]
author: B
draft: false
comment: true
---

我推荐的内容是什么。

我为什么会想到推荐给你。

它最打动我的地方是什么。

我希望你看到的部分是什么。

如果你看完了，我最想和你聊什么。
```

建议标签：

- `书`
- `小说`
- `电视剧`
- `电影`
- `音乐`
- `一起看`

### 4. 锻炼计划模板

category: 锻炼计划

### 5. 不好开口内容模板

适合慢一点说的话，建议默认加密。

```md
---
title: 有些话想慢一点说
published: 2026-06-10
description: 想认真表达的一些感受
category: 不好开口
tags: [情绪, 想说的话]
author: A
draft: false
comment: false
password: 你们约定的密码
passwordHint: 只有对方知道的提示
---

这件事是什么。

我的真实感受是什么。

我不是想指责你，我只是想让你知道什么。

我希望我们之后可以怎么做。

如果你看到这里，我最想对你说的一句话是什么。
```

建议标题：

- `先写下来给你看`
- `这件事我想认真说`
- `关于最近的一些感受`
- `有些话想慢一点表达`

### 6. 分类与标签固定规则

为了避免后面混乱，建议固定如下。

#### 分类只用这 4 个

- `日常`
- `推荐`
- `锻炼计划`
- `不好开口`

#### 标签按内容细分

示例：

- 日常：`生活` `心情` `见面` `旅行` `纪念日`
- 推荐：`书` `小说` `电视剧` `电影` `音乐`
- 锻炼计划：`周计划` `打卡` `跑步` `力量训练` `作息`
- 不好开口：`情绪` `感谢` `道歉` `期待` `想说的话`

## 二、站点改造实施清单

这份清单不要求改代码，而是说明接下来该按什么顺序推进。

### 第一阶段：先完成内容规则

目标：统一使用方法。

要做的事：

1. 确定两位作者的固定名字写法
2. 确定 4 个固定分类
3. 确定常用标签表
4. 确定哪些内容默认公开，哪些默认加密
5. 约定标题风格和更新频率

完成标准：

- 你们两个人都知道一篇内容该怎么写
- 同一类型内容不会写出完全不同的格式

### 第二阶段：建立基础栏目内容

目标：让站点先“活起来”。

建议先补这些内容：

1. `关于这个空间`
2. 3 到 5 篇日常记录
3. 2 到 3 篇推荐内容
4. 1 篇锻炼计划
5. 1 篇“不好开口”的测试内容

完成标准：

- 四个栏目都有实际内容
- 首页不再只是空壳博客

### 第三阶段：形成更新节奏

目标：让站点进入稳定使用状态。

建议节奏：

- 日常：随时更新
- 推荐：每周 1 次或有内容再发
- 锻炼计划：每周 1 次
- 不好开口：低频，按需要发布

完成标准：

- 连续使用 2 到 4 周
- 你们能感受到哪些栏目真正会常用

### 第四阶段：复盘是否需要页面层优化

目标：决定后续是否值得做视觉和结构调整。

重点观察：

1. 哪个栏目更新最多
2. 哪个栏目最有价值
3. 哪个栏目最容易弃用
4. 私密内容是否真的会长期使用
5. 是否需要更明显地区分两位作者

完成标准：

- 能明确知道后续是继续轻运营，还是需要做功能优化

### 第五阶段：后续可选优化方向

如果以后准备继续深入，再考虑这些：

1. 首页改成“双人空间首页”
2. About 页改成“关于我们/关于这个空间”
3. 推荐内容做更清晰的聚合展示
4. 锻炼计划做更明确的周期展示
5. 私密内容入口做更低曝光处理
6. 作者身份做更明显区分

## 三、最终执行顺序

最推荐现在就按这个顺序做：

1. 写一篇“关于这个空间”
2. 建立分类和标签规则
3. 各发 1 篇日常
4. 发 1 篇推荐
5. 发 1 篇锻炼计划
6. 测试 1 篇加密内容
7. 连续用两周再复盘

核心原则只有一句：

**先让内容跑起来，再决定要不要继续改页面。**
