What's wrong about the main method of a Java program is this ( ).
AJava program can have multiple main() methods

Suppose x, y, z are int. If (x == y) is true, which conditions below are ALWAYS true?
I. (x==y) && (y==z) && (x==z)
II. (x==y) || (y==z) && (x==z)
III. (x-y)*(x-z)*(y-z) == 0
AOnly I
BOnly II
CII and III only
DI, II, and III
✓ 答案：C2分
解析

III：x-y = 0，乘积为 0，必然成立

Complete the MainClass: ① printElements() ② deleteStartsWith(String prefix) ③ printStartsWith(String prefix)  (5 points)
5分
参考答案（三个方法填空）
import java.util.Iterator; import java.util.Vector; public class MainClass { Vector vector = new Vector(); public MainClass() { vector.add("country China"); vector.add("country America"); vector.add("country English"); vector.add("name Mary"); vector.add("name Linda"); vector.add("name Mike"); vector.add("name John"); } // ① 遍历输出所有元素 public void printElements() { Iterator it = vector.iterator(); while (it.hasNext()) { System.out.println(it.next()); } } // ② 删除以指定前缀开头的字符串（用 Iterator 避免 ConcurrentModificationException） public void deleteStartsWith(String prefix) { Iterator it = vector.iterator(); while (it.hasNext()) { String str = (String) it.next(); if (str.startsWith(prefix)) { it.remove(); // 安全删除 } } } // ③ 打印以指定前缀开头的字符串 public void printStartsWith(String prefix) { Iterator it = vector.iterator(); while (it.hasNext()) { String str = (String) it.next(); if (str.startsWith(prefix)) { System.out.println(str); } } } }
注意：删除时必须用 it.remove() 而非 vector.remove()，否则会抛出 ConcurrentModificationException。

Write a class to read a given text file (java.txt) and output contents to console. Hint: BufferedReader, FileInputStream, InputStreamReader. (10 points)
10分
参考答案
import java.io.*; public class ReadFile { public static void main(String[] args) { try { // 按题目提示：FileInputStream → InputStreamReader → BufferedReader FileInputStream fis = new FileInputStream("java.txt"); InputStreamReader isr = new InputStreamReader(fis); BufferedReader br = new BufferedReader(isr); String line; while ((line = br.readLine()) != null) { System.out.println(line); } br.close(); } catch (IOException e) { e.printStackTrace(); } } } // 输出: // This is a java test // 1 2 3 4 5
三层包装链：字节流 FileInputStream → 字符流 InputStreamReader（可指定编码）→ 带缓冲的 BufferedReader（支持 readLine()）。

Complete the singleton pattern for StudentInfo (only one instance can be created). ① field ② constructor ③ getSingletonInstance(). (10 points)
10分
参考答案（懒汉式单例）
public class StudentInfo { // ① 私有静态实例字段（延迟初始化） private static StudentInfo instance = null; private String name; private String ID; // ② 私有构造方法，防止外部 new private StudentInfo() { name = ""; ID = ""; } // ③ 公有静态工厂方法，第一次调用时创建实例 static public StudentInfo getSingletonInstance() { if (instance == null) { instance = new StudentInfo(); } return instance; } public String getName() { return name; } public String getID() { return ID; } }

// UserDTO.java public class UserDTO { private String userAccount; private String userPassword; public String getUserAccount() { return userAccount; } public void setUserAccount(String acc) { this.userAccount = acc; } public String getUserPassword() { return userPassword; } public void setUserPassword(String pwd) { this.userPassword = pwd; } } // IUserDAO.java（接口） public interface IUserDAO { boolean addUser(UserDTO user); } // OracleUserDAO.java（实现类） public class OracleUserDAO implements IUserDAO { public boolean addUser(UserDTO user) { // Oracle specific implementation return true; } } // RegisterForm.java public class RegisterForm { private IUserDAO userDAO; public RegisterForm(IUserDAO dao) { this.userDAO = dao; } public void register(String account, String password) { UserDTO dto = new UserDTO(); dto.setUserAccount(account); dto.setUserPassword(password); userDAO.addUser(dto); } }

Describe two design patterns that are used in Java GUI with class diagram. (20 points)

设计模式一：Observer Pattern（观察者模式 / 事件监听）
Java GUI 事件处理是 Observer 模式的经典应用。组件（如 JButton）是被观察者（Subject），事件监听器是观察者（Observer），按钮被点击时通知所有注册的监听器。
// UML 简图： // ┌──────────────────┐ ┌──────────────────────────────────┐ // │ JButton (Subject)│ │ <<interface>> ActionListener │ // │------------------│ │ + actionPerformed(ActionEvent) │ // │-listeners:List │─────uses──▶│ │ // │+addActionListener│ └───────────────┬──────────────────┘ // │+removeListener() │ │ implements // └──────────────────┘ ▼ // ConcreteListener // + actionPerformed(ActionEvent e) // 代码示例 class MyListener implements ActionListener { public void actionPerformed(ActionEvent e) { System.out.println("Button clicked!"); } } JButton btn = new JButton("Click"); btn.addActionListener(new MyListener()); // 注册观察者
设计模式二：Strategy Pattern（策略模式 / 布局管理器）
Java GUI 的 LayoutManager 是 Strategy 模式的实现。Container（容器）持有一个 LayoutManager 策略，可在运行时动态替换不同的布局算法。
// UML 简图： // ┌───────────────────┐ ┌──────────────────────────┐ // │Container (Context)│ ─────uses────▶│ <<interface>> LayoutMgr │ // │-------------------│ │ + layoutContainer(Cont.) │ // │-layout:LayoutMgr │ │ + preferredLayoutSize() │ // │+setLayout(mgr) │ └────────────┬─────────────┘ // │+add(Component) │ │ implements // └───────────────────┘ ┌────────────┼────────────┐ // FlowLayout BorderLayout GridLayout // 代码示例：运行时替换策略 JPanel panel = new JPanel(); panel.setLayout(new BorderLayout()); // 切换为 BorderLayout 策略 panel.setLayout(new GridLayout(3,3)); // 切换为 GridLayout 策略
两个模式对比：Observer 解决"事件通知"问题（谁在监听谁），Strategy 解决"算法切换"问题（如何布局）。两者都体现了针对接口编程的 OOP 原则。
